Style analysis for generated web work

Turn visual references into a usable design contract.

StyleTrace is a Node.js MCP server that inspects exact public references, extracts layout and style signals, and returns a prompt-ready brief you can build against and review against.

How it works

Reference in. Grammar out. Reviewable output back through the loop.

StyleTrace visits references with Playwright, captures page structure and evidence, derives ten aspect summaries, and packages the result into fields an agent can directly consume without guessing.

10 design grammar aspects such as hierarchy, spacing, imagery, navigation, motion, and form treatment
3 evidence modes: omit, inline, or file-backed sidecar output
1 shared review contract for checking generated HTML or image output against the extracted constraints
Step 01

Ingest exact references

Step 02

Step 03

Step 04

Review against the contract

Compare generated artifacts back to the same invariants and risks to make style review explicit, narrow, and screenshot-checkable.

Review contract

Make visual review concrete instead of subjective.

The output does not stop at inspiration. It includes style invariants to preserve, risks to avoid, composition blueprints for module rhythm, and viewport checks that keep hero hierarchy and CTA clarity intact.

Must match Preserve first-screen hierarchy, chrome density, module pacing, imagery treatment, and color architecture as hard constraints before polishing details.
Must avoid Avoid generic premium-tech flattening, unsupported motion claims, weak breakpoint assumptions, and turning every section into the same rounded card stack.
Uncertain areas Low-confidence inference stays visibly low-confidence, so forms, motion, or breakpoint behavior are treated as subordinate to stronger evidence.

Reference inputs

Use live pages, image references, screenshots, or bounded HTML.

Keep scope narrow. The system analyzes exactly what you point it at, which keeps the result auditable and makes downstream review easier to trust.

URLs

Public website references

Mixed inputs

Blend sources without widening scope

Pair screenshots, direct image assets, and small HTML fragments when a browsing session is not the only source of truth.

"type": "image" | "screenshot" | "html"
