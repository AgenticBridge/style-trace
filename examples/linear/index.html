Design grammar

From vague taste to named constraints that can survive implementation.

The output is organized around a ten-aspect grammar so downstream tooling can preserve the strongest signals first. Hierarchy, color architecture, spacing, imagery role, navigation logic, and responsive behavior stay legible as separate decisions instead of collapsing into one flattened style sentence.

Evidence-first summaries

Each aspect includes a concise summary, supporting observations, and confidence so teams know which signals are structural and which remain weaker inference.

Signature motifs

Motifs surface the memorable moves of a reference without treating every component detail as equally important.

Reconstruction directives

Operational guidance is already phrased for regeneration and review, not just for analysis.

Product flow

One downstream path for capture, synthesis, and comparison.

Style Trace keeps the MVP narrow. Public URLs go in. Structured design evidence comes out. That same result can power prompt construction, page regeneration, screenshot review, or a stricter contract for future scoring without splitting into separate analysis products.

Playwright-backed page capture Real layout and screenshot context, including module candidates, actions, navigation text, and responsive probing.
Module segmentation Ordered visual modules such as hero, proof, pricing, content, CTA, navigation, and footer.
Synthesis modes Single-site profiles stay faithful to one source while cross-site runs preserve shared patterns instead of category averages.
File-based evidence Optional sidecar JSON and capture paths keep the public result compact while still supporting audits.
