Evidence-first website style analysis for MCP workflows

Read a website’s design grammar before you try to rebuild it.

StyleTrace is a Node.js MCP server that opens public pages in Playwright, captures visual structure, and returns a compact grammar for hierarchy, spacing, navigation, imagery, and component treatment. It is built for original reinterpretation, not brand mimicry.

What the server actually extracts

  • StructureVisual modules, section rhythm, navigation footprint, footer behavior.
  • StyleTypography scale, contrast discipline, spacing density, component weight.
  • EvidenceObserved signals, confidence markers, and optional capture paths for auditability.
  • SynthesisA prompt-ready brief for building a new page without lifting branded assets or copy.
Approach

Capture first. Infer second. Preserve only what the page actually proves.

The pipeline stays narrow on purpose: validate exact public URLs, open them in an isolated browser context, capture structural and stylistic signals, segment the page into modules, and synthesize a design grammar that stays closer to evidence than taste.

  • Browser-level inspection

    Pages load in headless Chromium with desktop and mobile probing, so hierarchy and responsive behavior are derived from rendered output rather than source guesses.

  • Module-aware analysis

    Sections are turned into ordered visual modules such as navigation, hero, proof, pricing, content, and footer, preserving rhythm and emphasis instead of flattening everything into a summary sentence.

  • Ten-aspect grammar

    Output covers hierarchy, typography, color, spacing, imagery, states, navigation, motion, forms, and breakpoints, with confidence levels that keep weaker inferences from acting like hard facts.

Output

A design brief that downstream tools can use without inventing the rest.

StyleTrace returns structured JSON for single-site profiling or cross-site commonality. The result is meant to guide reconstruction and review: what must remain faithful, what should stay original, and where the model should stay cautious.

The server can keep evidence out of the public payload, inline it directly, or write it to a sidecar artifact. This makes it usable in tighter prompt contexts while still supporting audit trails when a generation needs to be checked against source material.

The synthesis layer produces style invariants, style risks, composition blueprints, and prompt-ready guidance. That keeps a redesign anchored to real composition signals instead of sliding toward a generic startup landing page.

Why it holds up

Useful when fidelity matters, especially when “similar” is too vague.

A homepage can borrow pacing, density, and composition without borrowing identity. StyleTrace is for that boundary: enough specificity to preserve the reference’s visual logic, enough restraint to avoid copying the reference itself.

  • For regeneration reviews

    Compare a generated page against captured hierarchy, navigation density, and module order with explicit rules about what must match and what must not be invented.

  • For design system research

    Use the extracted vocabulary to understand how public product sites organize emphasis, spacing, and interfaces across real pages instead of abstract style labels.

  • For original builds

    Keep branded copy, logos, imagery, and protected expression out of scope while still translating the reference into a practical build brief for a new artifact.
