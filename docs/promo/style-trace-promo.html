What StyleTrace actually extracts

Not raw DOM dumps. Not vague inspiration. A compact, evidence-first profile tuned for implementation and review.

Structure

Pages + sections

Tracks analyzed paths, page intent, and top-of-page section signals like hero, proof, features, pricing, and CTA.

Style profile

Tone, layout, components

Summarizes nav density, button treatment, typography, content width, card usage, proof modules, and pricing block style.

Reproduction cues

Actionable patterns

Captures hero media style, CTA pattern, button radius/emphasis, and commerce/proof presence for implementation reuse.

Artifact flow

Review-ready outputs

Produces source captures, compact JSON, markdown, and a regenerated HTML review artifact for human inspection.

How the review flow works

The promo assets are generated from the same review pipeline maintainers use when evaluating a real instance.

1

Run the MCP

Call analyze_website_style on public URLs and save the payload.

2

Capture source pages

Use Playwright to screenshot each analyzed page so reviewers can compare references directly.

3

Regenerate pure HTML

Use the MCP output to produce a review-only standalone HTML page that reflects the extracted style family.

4

Review the artifact set

Inspect manifest.json, the JSON result, the regenerated screenshot, and all source captures side by side.

Try it
